# Visual designers — interview guide

## Role context

Visual designers at RDC (Joy Baer, Nina Stackhouse) produce slide decks, infographics, ebooks, social cards, flyers, banners, brochures, and brand assets. Most work starts from a client brief or content document and involves translating written content into a visual layout within established brand guidelines.

Common work patterns:
- Receiving a content document or brief and building a visual layout from it
- Working from past examples ("make it look like this one, but updated")
- Going through multiple review rounds where feedback often covers copy changes, not just design
- Managing version control across rounds (v1, v2, v3)
- Writing design notes or alt-text descriptions for production handoff
- Interpreting vague or incomplete briefs and making judgment calls

AI's direct role in creating visual assets is limited. The strongest opportunities are in the written and organizational side of the work.

## Likely friction areas to probe

- **Brief interpretation**: Getting a vague or incomplete brief and having to figure out what's being asked before designing
- **Copy changes during review**: Text edits scattered across feedback after the design is done
- **Alt-text and design notes**: Writing visual descriptions for production or accessibility — repetitive
- **Version summaries**: Writing up what changed between design rounds
- **Feedback organization**: Interpreting and organizing feedback that sometimes contradicts itself
- **Deck copy suggestions**: Writing placeholder copy or section headers before the writer has provided final copy

## Question sequence

**Q3 — Deliverable mix** (buttons):
> "What types of design work do you spend the most time on?"

Options: Slide decks and presentations / Infographics and visual documents / Social media assets / Ebooks or long-form designed documents / Something else

**Q4 — Where time gets lost** (buttons):
> "Which part of a typical design project takes longer than it should?"

Options: Figuring out what the brief is actually asking before I can start / Going back and forth on copy changes during review / Managing feedback — organizing it, figuring out what to act on / Writing things up: version notes, design handoff docs, alt text / Something else

**Q5 — Drill-down** (buttons, branch on Q4 answer):

If "brief interpretation":
> "When a brief is vague or incomplete, what do you usually do?"
Options: Ask the PM to get more info from the client / Make my best judgment and flag assumptions / Look at past similar projects and adapt / Start with something and wait for feedback to correct course / Something else

If "copy changes during review":
> "When copy edits come in during a review round, what's the most annoying part?"
Options: There are too many small edits scattered across many slides / The feedback isn't specific enough to know exactly what to change / I have to reformat or reflow the design every time copy changes / Tracking what was changed and confirming it was addressed / Something else

If "managing feedback":
> "What makes design feedback hard to work with?"
Options: It's vague — "make it feel more premium" without specifics / It contradicts itself — different reviewers want different things / There's a lot of it and it's hard to organize / It arrives in different places — email, comments, calls, Teams / Something else

If "writing things up":
> "What written tasks come up most often in your design work?"
Options: Alt text or accessibility descriptions / Version summary notes (what changed in v2 vs v1) / Design handoff notes for production / Responding to client feedback in writing / Something else

**Q6 — Open-text moment** (see Step 4 in SKILL.md)

**Q7 — AI experience** (buttons):
> "Have you used Claude or any AI tools for work before?"

Options: Yes, fairly regularly / A few times, with mixed results / Once or twice but it didn't really stick / Not really

**Q8 — Barrier check** (buttons):
> "What would make an AI solution feel too complicated to bother with?"

Options: If it doesn't integrate with my design tools (Figma, PowerPoint, etc.) / If it requires a lot of setup / If the output still needs major cleanup / If it only helps with writing, not actual design / Something else

## Output guidance

Strong recommendation candidates for this role:
- A feedback consolidation workflow: paste all feedback from a review round, get an organized action list grouped by slide or section
- A version summary generator: describe what changed, get a clean summary to send to the PM or client
- A brief-to-questions workflow: paste a vague brief, get clarifying questions to ask the PM before starting
- A copy-fit helper: paste copy that's too long, give Claude the constraint, get a trimmed version
- An alt-text generator: paste descriptions of visual elements, get accessibility-ready alt text

Example "Where to start" prompt for the output document:
> "I'm a designer working on a slide deck. Here's all the feedback I received: [paste feedback]. Organize this into a clear action list, grouped by slide number, so I can work through it efficiently."
