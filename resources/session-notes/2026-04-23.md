## Recurring pattern: claude binary not in PATH for automated contexts

`claude` lives at `/home/joe/.local/bin/claude` (symlink) and `/usr/local/bin/claude`. Neither systemd services nor cron jobs inherit the user's `$PATH`, so any automated context calling `claude` bare will fail with `command not found`.

**Fixes applied today:**

- `bodkin-daily-log.service` — added `Environment=PATH=/home/joe/.local/bin:/usr/local/bin:/usr/bin:/bin` to the `[Service]` block, then `systemctl daemon-reload`
- Crontab — added `PATH=/home/joe/.local/bin:/usr/local/bin:/usr/bin:/bin` at the top to cover task-spur and all other cron scripts
- `vault-gardening-auto.sh` — replaced bare `claude` with `/usr/local/bin/claude` (absolute path)

**Rule going forward:** any new script or service that calls `claude` must either use the absolute path or set PATH explicitly. Systemd services need `Environment=PATH=...` in the service file; cron jobs need a `PATH=` line at the top of the crontab or within the script.
