# Azure SMB content — language and messaging principles

This skill encodes the SMB-specific language principles, tone guidelines, copy conventions, and messaging frameworks from Microsoft's SMB Integrated Marketing team and the official FY26 H1 Messaging & Positioning Framework (MPF). Apply whenever writing Azure content targeting small and medium business audiences.

---

## Workflow

### Step 1: Identify the audience segment
Read `references/audience-segments.md`. Confirm which SMB segment is being targeted. Default to Classic SMB if unspecified. Do not write until the segment is clear.

### Step 2: Apply language rules
Read `references/language-rules.md`. Flag any enterprise terminology in source materials before drafting. When a brief contains enterprise-first positioning, adapt it proactively rather than copying it through.

### Step 3: Set tone and voice
Read `references/tone-and-voice.md`. Lead with business outcomes, not product names or feature lists.

### Step 4: Consult messaging pillars if needed
If writing a hero statement, positioning paragraph, or any copy that needs to reflect approved Microsoft positioning, read `references/messaging-pillars.md`.

### Step 5: Apply format-specific guidance
Based on the deliverable type, read the appropriate template file:
- Infographic or use case writing → `templates/infographic.md`
- E-book → `templates/ebook.md`
- Social cards → `templates/social-card.md`

### Step 6: Select CTA
Read `references/cta-options.md` for the correct CTA by segment and play.

### Step 7: Source customer stories (if applicable)
Read `references/customer-stories.md` for filtering and selection criteria.

### Step 8: Apply photography direction (if applicable)
Read `references/photography.md` when providing image direction to a designer.
