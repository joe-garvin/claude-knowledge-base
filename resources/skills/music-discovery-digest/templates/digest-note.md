# Digest note template

Use this structure for every weekly digest note. Populate all sections; omit a section only if there are genuinely zero candidates (note the omission).

---

```
---
created: "YYYY-MM-DD"
updated: "YYYY-MM-DD"
source: "[comma-separated sources used]"
week: "YYYY-WNN"
tags:
  - music-discovery
  - digest
---

# YYYY-WNN — Weekly digest
*Week of [Month D, YYYY] · Sources: [list sources]*
*Cross-referenced against local library and Spotify playlists. Known artists flagged accordingly.*
[Optional note if a source was unavailable or the digest was run early]

## New discoveries

**[Artist] — [Release Title]**
Label: [Label] · Genre: [genre tags] · [Released: Month D, YYYY if notable]
[2–4 sentences: what it sounds like, why it matches, what makes it a discovery. Avoid filler. Trust the reader.]
[Source link if available: Boomkat URL or RA review URL]
Preview: [YouTube search URL]

[Repeat for each discovery — target 2–4]

## New releases from artists you know

**[Artist] — [Release Title]** *(new release / recent release)*
Label: [Label] · Genre: [genre tags] · Released: [date]
[2–4 sentences: same format as above, but note the connection to the known collection.]
[Source link]
Preview: [YouTube search URL]

[Repeat — target 1–3]

## Wildcard

**[Artist] — [Release Title]**
Label: [Label] · Genre: [genre tags]
[2–4 sentences: explicitly note why this is a wildcard — what's adjacent or outside the usual profile, and what earns its place anyway.]
[Source link]
Preview: [YouTube search URL]

[One wildcard per digest]

## Also noted

- **[Artist] — [Release]** ([Label]) — [one sentence: why it's worth tracking but didn't make the main picks]
- [Repeat]

---

*[Optional closing note: source gaps, thin week, follow-up items]*
```
