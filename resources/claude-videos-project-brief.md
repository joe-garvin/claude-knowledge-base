## Claude training video series — project brief

### Overview

A series of short screen-recording videos designed to help RDC team members get more out of Claude in their day-to-day work. The goal is practical orientation grounded in real utility — each video should show people something they can act on, not survey features for their own sake. Target audience is a broad mix of RDC roles: writers, project managers, and others who may have varying levels of Claude familiarity.

The guiding principle across all videos: show how Claude helps people do what they already do, faster and more consistently. Enthusiasm for AI tools is not assumed. Reduced friction and tangible time savings are the value proposition.

### Tone and format goals

- Conversational narration — the tone should feel like a knowledgeable colleague showing you something useful, not a training module
- Each video should establish the practical benefit early, before getting into how-to
- Technical depth calibrated to a mixed writer/PM audience: enough to be useful, not enough to intimidate
- Target length: 8–10 minutes per video; enough room to establish context, demonstrate the workflow, and close with clear takeaways without rushing

### Video ideas

#### Skills: working with pre-built workflows

**Purpose:** Show how pre-built Claude skills let you trigger structured, repeatable workflows without having to think about prompting. The value is consistency and speed — someone already did the setup work, and now you just use it.

**Core angle:** You don't have to build anything. Someone built it, you install it, and it runs when you need it.

**Content outline:**

1. What a skill is and how it differs from a regular prompt — the key distinction is that a skill encodes a full workflow, not just an instruction
2. What kinds of tasks are good candidates for skills — recurring work, multi-step processes, anything where you find yourself re-explaining the same context
3. How to install a skill someone else has built and shared
4. How to trigger a skill — phrase-based or slash command
5. A live demo using an actual RDC skill relevant to writers or PMs (e.g., meeting debrief, task triage)
6. Brief coda: if you want to build your own skill, that's possible — could be a future video if there's interest

**Notes:** RDC-specific skill examples will make this land better than generic demos. Use a skill with an obvious, felt benefit for the demo — something that saves noticeable time on a task people do often.

------

#### Projects and persistent context

**Purpose:** Show how Claude projects eliminate the setup tax of starting from scratch every time. A project carries context forward — the background, the style, the files — so every new chat inside it already knows what it needs to know.

**Core angle:** Stop re-explaining yourself. Build the context once, work from it indefinitely.

**Content outline:**

1. The problem this solves — what it feels like to start a cold chat vs. working inside a project that already knows your context
2. What lives at the project level: instructions, files, shared context
3. How to set up a project for a real workstream — live demo building one from scratch
4. Using Claude to generate project instructions: ask Claude to interview you, gather what it needs, and produce instructions you can paste in
5. Using Claude to generate project context files — markdown notes that function as useful background (style guides, client context, recurring reference material)
6. What a chat inside a well-configured project looks and feels like vs. a cold chat

**Notes:** The interview technique — asking Claude to ask you the right questions before generating instructions or context — is worth demonstrating explicitly. It reframes setup from "I have to know what to write" to "Claude will help me figure out what it needs."

------

#### Memory and continuity

**Purpose:** Help people understand how Claude's memory actually works — and give them practical habits for maintaining continuity across chats and projects without losing ground.

**Core angle:** Claude doesn't remember between chats by default. Here's how to work with that instead of being surprised by it.

**Content outline:**

1. How memory works in Claude — what carries over and what doesn't, why context disappears between chats
2. The handoff document workflow — at the end of a productive chat, ask Claude to generate a summary of what was accomplished, what decisions were made, and what comes next; paste that into the next chat to pick up where you left off
3. Generating specs and briefs that Claude can follow — documents that front-load enough context that any new chat can get oriented quickly
4. The interview technique: asking Claude to interview you to gather enough information to produce an optimized brief, spec, or prompt — useful when you're not sure what context Claude actually needs
5. Practical habits: what to include at the start of a chat, how to name and organize project files for reuse, when to start a new chat vs. continuing an existing one

**Notes:** The handoff document is probably the single most immediately useful technique here for a PM or writer who's already using Claude regularly. Lead with it. The interview technique bridges naturally into the projects video — consider cross-referencing.

------

#### Claude product orientation

**Purpose:** Give team members a clear mental model of the Claude product landscape — what's available, what each tool is actually for, and which roles get the most out of each. The goal is to reduce confusion, not create a features tour.

**Core angle:** Here's which tool is yours, based on what you actually do.

**Content outline:**

1. Claude web UI vs. Claude desktop app — practical differences, why the desktop app is recommended for regular work
2. Claude Chat vs. Co-work — when each makes sense
3. Claude Code — what it is, who it's actually for, and a genuine demystification (not "don't worry, it's not scary" — show concretely why a writer or PM wouldn't need it, and what kind of person would)
4. Claude Design — what it does and who benefits
5. A simple decision framework: given your role and the kind of work you do most, here's where to start

**Notes:** This video is probably the most broadly useful for someone who's new to Claude or confused about what they have access to. It could function as a series intro for some viewers. Keep the Claude Code section honest — the goal is accurate framing, not reassurance.

### Production setup

**Platform:** Mac (personal MacBook Pro, logged into work Claude account).

**Primary recording tool: Screen Studio** ($9/month)

- Cursor highlighting, click indicators, and auto-zoom-to-cursor handled automatically
- Polished output without manual editing overhead
- Live narration while recording — keeps pacing natural, reduces post-production time

**Audio:** Personal microphone. No changes needed.

**Video:** Webcam available. Face-cam overlay optional — consider using it for openings and closings, skip it during screen walkthroughs. Evaluate after first recording.

**If additional editing is needed:** ScreenFlow for annotation overlays, callouts, or precise trim editing. Not needed to start.
