# Avoiding AI writing — RDC resources

**The guide** is organized around RDC's actual content types — each format (blog, video script, social, e-book, infographic, etc.) gets its own brief note on which AI tells matter most for that format. The editorial workflow reads as a numbered process rather than a research summary. The closing checklist is copy-paste ready for any writer to use before submitting a piece.

**The word list** includes client-specific notes at the bottom — what matters for Databricks content (technical precision, name the integrations) is different from what matters for City of Seattle content (plain language, drop the jargon). The Tier system is explained in accessible terms without any of the academic/linguistic framing from the original.

**The tools doc** leads with the custom instruction prompts, which are immediately actionable for any writer using any of the three chatbots. There are four ready-to-paste editorial prompts (general, Microsoft-specific, Databricks-specific, City of Seattle-specific). The avoid-ai-writing skill section explains GitHub and the SKILL.md in plain English — "go to this URL, click this file, copy the contents, paste it into Claude" — and flags the more advanced setup as requiring someone with developer tools.
