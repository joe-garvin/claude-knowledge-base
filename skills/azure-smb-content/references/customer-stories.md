# Customer stories guidance

## Selection criteria

- Prioritize SMB customer stories when customizing or supporting content
- Select stories published within the past two years
- Filter by: Azure + relevant product(s); organization size 50–999 employees; partners if applicable
- Stories from businesses similar in size and industry to the target audience build the most trust

## A note on ISV / vendor stories

Customer stories from companies that *built products on Azure* (ISVs, software vendors) are not the same as stories from SMB end-users. ISV stories read as developer case studies, not "businesses like yours" proof points. Outcomes tend to be technical (API call volume, model efficiency) rather than business-oriented. Flag ISV stories before using them in BDM-facing content and check whether the underlying business outcome can be reframed in end-user terms.

If the use case maps to an ISV story, ask the client whether end-user customer stories are available before defaulting to the ISV version.

## Confirm before publishing

Always confirm with the client whether named customer stories are approved for external use, or whether they need to be anonymized (e.g., "a growing legal software platform").
